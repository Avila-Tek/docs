# Conceptos básicos

Esta sección reúne los **conceptos básicos de código** que necesitamos manejar en Landscapes para trabajar mejor con Lovable, incluso sin tener un perfil técnico.

La idea no es convertirnos en developers, sino entender el lenguaje base del producto para poder:

- Pedir cambios con más claridad,
- Detectar mejor dónde está un problema,
- Tomar decisiones con más contexto,
- Y reducir errores al tocar partes sensibles del proyecto.

Aquí documentamos conceptos simples y prácticos, aterrizados al contexto real de Landscapes: proyectos rápidos, trabajo guiado por prompts y cambios que pueden impactar frontend, backend, base de datos y producción.
