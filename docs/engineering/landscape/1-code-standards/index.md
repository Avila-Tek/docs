# Estándares de código

Esta sección define los **estándares de ingeniería** que usamos en Landscapes para mantener nuestra prioridad #1: **velocidad**.

La idea no es “hacer código perfecto”, sino tener reglas claras que nos permitan:

- Tomar decisiones rápido (menos discusión, más ejecución),
- Mantener la calidad bajo control (menos bugs y menos retrabajo),
- Hacer cambios pequeños y predecibles (más fácil de revisar y mantener).

Aquí documentamos estándares agnósticos a la herramienta: aplican igual si el trabajo lo hace una persona o una IA (Lovable u otra).
